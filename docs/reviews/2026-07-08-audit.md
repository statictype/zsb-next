# Component audit ‚Äî `about/` + `partners/` page recipes

**Scope:** `src/app/(site)/about/{page.tsx,page.recipe.ts}`,
`src/app/(site)/partners/{page.tsx,page.recipe.ts}`.

**Raw-value policy applied** (quoted from `src/design-system/preset.ts:16-28`):
`strictTokens` is ON; every value is a token or a deliberate `[bracket]`.
Brackets are **debt to justify or remove, not blessed art direction**; the rule
is *"second occurrence ‚Üí token"* and *"snap to nearest token, accept a small
visual delta"* ‚Äî keeping a raw literal only when the exact number is
load-bearing (hairline px, %/vw centring, proportional geometry).

These two pages are siblings by construction (editorial spread ‚Üí dark band ‚Üí
light "why/curator" block ‚Üí CTA/letter) and the recipes have drifted into
parallel copies rather than shared vocabulary. Most findings are cross-file
duplication that the DS already has a home for.

## Detailed findings (most-actionable-first)

### DS-1 ¬∑ `statementInner` / `partnerCtaInner` re-declare the section rail ‚Äî no-op
- **file:** `about/page.recipe.ts:110-124`, `partners/page.recipe.ts:127-136`
- **axis:** ds-hygiene ¬∑ **risk:** no-op ¬∑ **effort:** trivial
- **smell:** both slots open with `maxWidth: 'maxWidth'; marginInline: 'auto';
  paddingInline: 'gutter'` ‚Äî the *exact* body of `layerStyles.sectionInner`
  (`tokens.ts:502-504`) ‚Äî then add their own grid/flex. The `inner` slot in the
  same two files already does `{ layerStyle: 'sectionInner' }`, so this is an
  inconsistency within one file.
- **evidence:** `sectionInner` layerStyle = `{ maxWidth: 'maxWidth',
  marginInline: 'auto', paddingInline: 'gutter' }`.
- **recommendation:**
  - `statementInner: { layerStyle: 'sectionInner', lg: {‚Ä¶}, '2xl': {‚Ä¶} }`
  - `partnerCtaInner: { layerStyle: 'sectionInner', display: 'flex',
    flexDirection: 'column', alignItems: 'center', justifyContent: 'center',
    textAlign: 'center' }`
- Zero visual change; single source for the rail.

### DS-2 ¬∑ `statement._before` reinvents `borderTop: 'hairline'` ‚Äî no-op
- **file:** `about/page.recipe.ts:100-109`
- **axis:** ds-hygiene / adversarial (two models, one concern) ¬∑ **risk:** no-op
  ¬∑ **effort:** trivial
- **smell:** `statement` paints a top rule via a full pseudo-element
  (`position: relative` + `_before { inset:[0 0 auto]; height:[1px]; background:
  divider }`). The sibling page paints the identical top rule with one line ‚Äî
  `partnerCta: { borderTop: 'hairline' }` (`partners/page.recipe.ts:124-126`).
- **evidence:** `hairline` border = `1px solid {colors.divider}`
  (`tokens.ts:132`); under the light ground the `_before`'s `background:
  divider` and the hairline's `--colors-divider` resolve to the *same*
  `gray.200` (`ground.ts:14`) ‚Äî so the two produce an identical line.
- **recommendation:** replace the `_before`/`position: relative` with
  `borderTop: 'hairline'`; drops two brackets (`[0 0 auto]`, `[1px]`) and one
  pseudo-element. Matches the partners CTA. (`statementAside`'s `sticky` does not
  need the parent `position: relative`.)

### DEAD-1 ¬∑ `pillarHead` is a wrapper around a single child ‚Äî no-op
- **file:** `about/page.tsx:80-82`, recipe `about/page.recipe.ts:84`
- **axis:** dead-code / adversarial (redundant nesting) ¬∑ **risk:** no-op ¬∑
  **effort:** trivial
- **smell:** `pillarHead` is `{ display: 'flex', alignItems: 'baseline', gap:
  'md' }` but wraps exactly one element ‚Äî the `<h2>` title. The flex/gap/baseline
  only make sense in the partners twin (`whyPillarHead`,
  `partners/page.tsx:114-117`) which holds *two* children (`whyPillarNum` +
  title). This is the numbered-head pattern copied minus the number.
- **recommendation:** drop the `pillarHead` wrapper (render the `<h2>` directly),
  or delete the slot and keep the `<div>` only if a future number is planned.

### DS-3 ¬∑ duplicated cover+grayscale image block ‚Äî small-visual (layerStyle extract)
- **file:** `about/page.recipe.ts:47-51` (`placeImageImg`),
  `partners/page.recipe.ts:57-61` (`eventImageImg`),
  near-variant `:76` (`whySculptureImg`)
- **axis:** ds-hygiene (reuse) ¬∑ **risk:** small-visual ¬∑ **effort:** small
- **smell:** `placeImageImg` and `eventImageImg` are byte-identical ‚Äî
  `{ objectFit: 'cover', background: 'gray.900', filter: '[grayscale(100%)
  contrast(1.05)]' }`. `whySculptureImg` is the same shape with `gray.200` bg and
  no `contrast`. Grep confirms the `grayscale(100%) contrast(1.05)` filter exists
  in exactly these two files ‚Äî the "second occurrence ‚Üí token" trigger.
- **recommendation:** extract a `coverMono` layerStyle
  (`objectFit: 'cover'; filter: '[grayscale(100%) contrast(1.05)]'`) with the
  backdrop as the call-site delta; the filter bracket stays (functional filter)
  but centralised. `whySculptureImg` composes it and overrides `background`/drops
  `contrast` (accept the small delta, or make contrast the variance point).

### DS-4 ¬∑ `prose` textStyle re-typed + undocumented line-height ramp ‚Äî small-visual
- **file:** `partners/page.recipe.ts:44-49` (`eventBody & p`), `:114-119`
  (`whyPillarBody`)
- **axis:** ds-hygiene (reuse + magic ramp) ¬∑ **risk:** small-visual ¬∑ **effort:**
  small
- **smell:** both blocks are `{ fontFamily: 'body', fontSize: 'base', lineHeight:
  { base: 'body', md: 'loose' }, color: 'body' }`. Strip the breakpoint and that
  is exactly the `prose` textStyle (`tokens.ts:491-493`). The `{ base: 'body',
  md: 'loose' }` ramp appears *only* in these two spots (grep) with no documented
  rationale ‚Äî a magic ramp. Note the about twin already does this right:
  `letterBody & p` uses `textStyle: 'leadLarge'` (`about/page.recipe.ts:189`).
- **recommendation:** set the `<p>` typography to `textStyle: 'prose'` and drop
  the `md: 'loose'` bump (small-visual ‚Äî slightly tighter leading on the event
  intro at md+), or, if the looser measure is intentional editorial, add it to
  `prose` / a new `proseLoose` textStyle and document why. Either way one source,
  no bare ramp.

### DS-5 ¬∑ display-label treatment re-typed 3√ó ‚Äî small-visual (new textStyle)
- **file:** `about/page.recipe.ts:85-92` (`pillarTitle`), `:168-175`
  (`authorName`), `partners/page.recipe.ts:106-113` (`whyPillarTitle`)
- **axis:** ds-hygiene (reuse) ¬∑ **risk:** small-visual ¬∑ **effort:** small
- **smell:** all three share `{ fontFamily: 'display', textTransform:
  'uppercase', lineHeight: 'tight', letterSpacing: 'tight' }` and differ only in
  `fontSize` and `color`. No existing textStyle matches (`cardTitle` uses
  `lineHeight: 'heading'`, not `tight`; `metaLabel` is body-font) ‚Äî so this is a
  genuine gap, not a missed reuse.
- **recommendation:** add a `labelDisplay` textStyle carrying the 4 shared props;
  each slot keeps only its `fontSize`/`color`. Consistent with the project's
  "typography-only textStyles, margins/size at the call site" rule
  (`tokens.ts:430`). (In line with the endorsed aggressive-normalisation
  direction ‚Äî a DS addition, not a call-site hack.)

### DS-6 ¬∑ raw px max-widths not snapped to `measure` ‚Äî small-visual
- **file:** `partners/page.recipe.ts:151` (`partnerCtaBody maxWidth:[460px]`),
  `partners/page.tsx:98` (`SectionHeading ‚Ä¶ css({ maxWidth: '[700px]' })`),
  `about/page.recipe.ts:137` (`statementByline maxWidth:[240px]`)
- **axis:** ds-hygiene ¬∑ **risk:** small-visual ¬∑ **effort:** trivial
- **smell:** three bracketed px max-widths where the DS has a prose-measure token
  (`measure` = `60ch`, `tokens.ts:158`).
- **recommendation:**
  - `partnerCtaBody [460px]` ‚Üí **snap to `measure`** (‚âà same, small-visual).
  - `SectionHeading [700px]` ‚Üí heading measure runs wider than body `measure`; if
    it recurs, add a `measureWide` size token, else **justify** the bracket inline.
  - `statementByline [240px]` ‚Üí **keep, with a why-comment**: it is load-bearing
    geometry ‚Äî it matches the portrait's mobile intrinsic width
    (`Figure sizes="(max-width:1023px) 240px, 340px"`, `about/page.tsx:104`).
    Snapping it alone would desync frame vs. image request.

### DS-7 ¬∑ `partnerCtaHeading` one-off display clamp ‚Äî small-visual
- **file:** `partners/page.recipe.ts:138-144`
- **axis:** ds-hygiene ¬∑ **risk:** small-visual ¬∑ **effort:** small
- **smell:** `fontSize: '[clamp(48px, 8vw, 106px)]'` is a lone bracketed display
  clamp. It is drift from the `pageTitle` textStyle family
  (`fontSize: 'clamp(40px, 4.75vw, 100px)'`, `lineHeight: '1'`, `tokens.ts:445`),
  which this heading visually echoes (`lineHeight: 'display'` = `1`).
- **recommendation:** if the CTA heading is *not* uppercase, it can't take
  `pageTitle` wholesale ‚Äî instead promote the clamp to a `fontSizes` token (e.g.
  a `6xl`/`hero` step) so CTA and page titles draw from one ramp, or justify the
  bracket as a deliberate one-off with a comment. No silent bracket.

### STRUCT-1 ¬∑ pillars grid & why grid are a duplicated mini-component ‚Äî visual-risk
- **file:** `about/page.tsx:77-86` + recipe `:60-96`; `partners/page.tsx:111-121`
  + recipe `:78-120`
- **axis:** structure ¬∑ **risk:** visual-risk ¬∑ **effort:** involved
- **smell:** both pages hand-roll the same shape: a `1fr ‚Üí 1fr 1fr` grid of
  `<article>` cells, each `flex column gap-md` with a head + display-uppercase
  title + prose body, cells separated by `hairline` borders. `about` has 9 slots
  for it (`pillarsGrid/pillar/pillarHead/pillarTitle/pillarBody`), `partners` 6
  (`whyGrid/whyPillar/whyPillarHead/whyPillarNum/whyPillarTitle/whyPillarBody`).
  The only real differences: partners numbers each cell (`pad(i+1)`), uses
  `nth-child(odd/even)` border logic vs about's `first/last-child` +
  `marginBlock:'4xl'`, and title colour tracks the ground (`highlight` on dark vs
  `heading` on light). No shared component exists (`src/components` has none).
- **evidence:** the two recipes are ~80% identical; DS-5 (title) and DS-4 (body)
  already show the leaf styles want to be shared.
- **recommendation:** extract a `PillarGrid` component (co-located recipe) owning
  grid + cell + head + title + body, with props for `numbered?: boolean` and the
  border rhythm; title colour inherits from the section ground (as
  `SectionHeading` already does). Each page passes data + the two deltas. Fold
  DS-4/DS-5 into it.
- **Left for a human eyeball** ‚Äî the border/margin schemes genuinely diverge
  (about's `marginBlock:'4xl'` + `first/last` vs partners' `nth-child`), so the
  extraction needs a visual pass, not a mechanical swap.
